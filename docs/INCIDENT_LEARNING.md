# Incident Learning And QIP

Incident learning is linked to QIP, but it is not the same thing.

An incident review or learning response can identify:

- system hazards
- contributory factors
- weak barriers
- improvement actions
- themes that need measurement

It becomes a QIP only when there is:

- a specific aim
- outcome, process, and balancing measures
- a change idea
- PDSA or equivalent iterative testing
- follow-up data

## PSIRF Position

PSIRF is a UK NHS patient-safety incident response framework. It should be a UK module, not the universal core.

Use PSIRF-aware prompts to:

- keep engagement with affected patients, families, and staff visible
- separate learning response from improvement action
- avoid simplistic single-cause thinking
- prompt proportionate response and local governance checks

Do not use this kit to:

- process identifiable incident narratives
- decide whether an event is reportable
- replace local patient-safety team judgement
- write formal PSIRF reports without approved local governance

## Public Product Decision

Keep incident learning as a companion module:

- `QIP Guru` remains the public product concept
- `QIP Guru Kit` remains the main repo/toolkit
- `Incident Learning Pack` can be a later extension
- UK PSIRF content lives under the UK profile
- other countries can add their own incident-learning modules
