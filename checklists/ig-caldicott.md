# IG And Caldicott Checklist

This checklist is an educational scaffold. It does not replace local IG, Caldicott, audit, QI, or clinical governance review.

## Before Data Collection

- [ ] The project purpose is clear and proportionate.
- [ ] The local QI, audit, or governance route has been confirmed.
- [ ] The data source owner has agreed the extraction route.
- [ ] Minimum necessary data fields have been listed.
- [ ] Direct identifiers are excluded unless there is explicit local approval.
- [ ] Free-text fields are excluded or reviewed through an approved route.
- [ ] The storage location is approved for the data type.
- [ ] Access is limited to named people with a project role.
- [ ] Retention and deletion dates are recorded.
- [ ] Small-number and rare-combination disclosure risk is considered before sharing outputs.

## Caldicott-Style Prompts

- Can the purpose be achieved without patient-identifiable data?
- Can the dataset be reduced further?
- Who needs access, and why?
- What is the lawful and local governance basis?
- How will outputs be checked before wider sharing?
