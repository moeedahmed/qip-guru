# Security Policy

QIP Guru Kit is a local educational scaffold, not a patient-data processing service.

## Supported Versions

The project is still a public-release candidate. No published support window exists yet.

## Reporting Issues

Use a private maintainer channel for security issues until the public repository has a formal advisory route. Do not include real patient, staff, organisation-sensitive, incident-identifiable, credential, or secret material in a report.

A useful report includes:

- the affected command or file
- the installed version or commit
- a minimal synthetic reproduction
- expected and actual behaviour
- whether any generated output was created

## Data Safety Scope

For clinical, information-governance, Caldicott, audit, ethics, or publication boundaries, see `SAFETY.md`. The de-id scanner is assistive pattern matching only and is not reliable de-identification for real patient data.
